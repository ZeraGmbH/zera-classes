The sequence of adjustment commands is important.

  • Init:

    The init command must be used before starting the range adjustment.
    For e.g. CALCULATE:ADJ1:INIT UL1,250V; This makes a clear start for the UL1-250V adjustment. It doesn't change the EEPROM and already calculated adjustment data for other channels/ranges.

  • Amplitude:

    This command calculates the amplitude correction value and adds one amplitude correction node.
    For e.g. CALCULATE:ADJ1:AMPLITUDE UL1,250V,199.9865998284; As long as INIT command isn't called again, all old nodes are maintained.

  • Gstatus:

    Once all gain correction nodes for this range are set, the GSTATUS command can be used to set the status of gain correction.
    For example, CALCULATE:ADJ1:GSTATUS UL1,250V,128; The status value '128' means this range is adjusted.

  • Phase:

    This command calculates the phase correction value and adds one phase correction node.

  • Pstatus:

    Once all phase correction nodes for this range are set, the PSTATUS command can be used to set the status of phase correction.

  • Offset:

    This command calculates the offset correction value and adds one offset correction node.

  • Ostatus:

    Once all offset correction nodes for this range are set, the OSTATUS command can be used to set the status of offset correction.

  • Computation:

    The 'CALCULATE:ADJ1:COMPUTATION 1' command computes coefficients for all channels and all ranges from the nodes previously set.

  • Storage:

    With 'CALCULATE:ADJ1:STORAGE 1' the computed adjustment data is stored into EEPROM.
    Ideally, use this command at the end, when all channels and ranges are adjusted.

  • Send:

    The command CALCULATE:ADJ1:SEND forwards the command to the specified port.
    For e.g. CALCULATE:ADJ1:SEND? 6307,sens:mmod AC\;; The command 'sens:mmod AC;' is forwarded to port 6307.
